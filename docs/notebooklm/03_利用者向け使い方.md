# HDM ToDoアプリ 利用者向け使い方

対象: 齊藤・伊江さん・廣瀬さん  
目的: 実際に使う人が迷わず操作できるようにする

## 最初に覚えること

このアプリでは、作業をしたら必ずログを残します。

ログは長く書く必要はありません。  
短くても、作業したこと・困っていること・完了したことが残ればOKです。

## 毎日の使い方

1. アプリを開く
2. 自分の名前を選ぶ
3. 担当タスクを確認する
4. 作業したらログを書く
5. 進捗率を更新する
6. 完了したら完了にする

## 自分の名前を選ぶ

ダッシュボードの上部で、入力する人の名前を選びます。

選べる名前:

- 齊藤
- 伊江さん
- 廣瀬さん

名前を選ぶと、その人が入力中の状態になります。

## 担当タスクを見る

ダッシュボードでは、自分に関係するタスクが確認できます。

表示されるもの:

- 自分が担当しているタスク
- 全員で担当するタスク
- 管理者の場合は全タスク

## 現場ログを入力する

ダッシュボード上部の現場入力欄を使います。

入力するもの:

- 担当タスク
- 進捗率
- ログ内容

入力したら「現場ログ保存」を押します。

## ログに書く内容

ログには、作業の事実を書きます。

書くとよい内容:

- 今日やったこと
- どこまで終わったか
- 残っていること
- 困っていること
- 相談したいこと
- 必要な備品
- 完了報告

## ログ例

```text
車両初期傷登録を5台分進めました。残りは明日確認します。
```

```text
ガラス撥水コーティングを3台実施しました。洗車後に進める流れで問題なさそうです。
```

```text
コーンの破損が複数あります。購入が必要か確認お願いします。
```

```text
トレーラー清掃完了。砂とゴミは片付けました。
```

## 進捗率の目安

0%:

- まだ開始していない

25%:

- 少し進めた
- 作業開始済み

50%:

- 半分ほど完了

75%:

- かなり進んでいる
- 残り確認・仕上げ

100%:

- 完了

## 完了したとき

完了したら「完了」を押します。

必要に応じて、ログに完了内容を書きます。

例:

```text
環境整備day完了。草刈り、ゴミ回収、コーン整理、トレーラー清掃まで実施しました。
```

## 相談が必要なとき

止まっている理由をログに残します。

書くとよい内容:

- 何ができていないか
- なぜ止まっているか
- 何を判断してほしいか
- いつまでに判断が必要か

例:

```text
コーンバーが不足しています。全部購入するか、一部配置を変更するか判断が必要です。
```

## 写真や資料を残す

必要な場合は、タスク詳細から写真や資料を添付します。

使う場面:

- 作業前後の写真
- 備品の破損状況
- 完了状態の写真
- 報告資料

## 齊藤が見るところ

齊藤は管理者として、以下を確認します。

- 全体の進捗
- 期限が近いタスク
- 期限を過ぎたタスク
- 相談が必要なタスク
- 完了報告
- AI管理コメント
- 齊藤向け報告文

## 伊江さんが見るところ

主に確認するもの:

- ガラス撥水コーティング
- 車両コーティング報告
- 全員タスク

やること:

- 作業した台数をログに残す
- 残り台数を把握する
- 洗車後に進めやすい流れを考える
- 必要な相談を残す

## 廣瀬さんが見るところ

主に確認するもの:

- 環境整備day
- 環境整備内容報告
- 全員タスク

やること:

- 草刈り・ゴミ・コーン・トレーラー清掃の進め方を考える
- 作業順番や役割分担を考える
- 必要備品を確認する
- 改善提案をログに残す

## 注意点

- 作業したらログを残す
- 完了したら完了にする
- 遅れそうなら早めに相談する
- 口頭だけで終わらせず、アプリにも残す
- カレンダーとあわせて確認する

## 共有時の一言

このアプリは、誰かを管理するためだけのものではありません。  
店舗全体で今月の目標を達成するために、作業を見える化し、成果を残すためのものです。
